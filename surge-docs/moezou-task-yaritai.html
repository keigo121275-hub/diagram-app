もえぞうタスク｜やりたいこと・方向性

コンサル・案件向けタスク管理の整理資料 · 2026年3月28日

目的：「チームの工数管理」ではなく、コンサル・クライアントワークに寄せたタスクの見え方・運用にそろえていく。納期は大事だが、優先の語彙も揃えたい。

1. いまの方針（合意していること）

  • 用途はコンサルメイン。従来の「チーム依頼フロー」中心のUIは外した。
  • Firebase プロジェクト moezoutask と既存 TeamBoard（task-91d08）は別物のまま運用。
  • タスクには追加した人の表示名（creatorName）を保存し、カード・詳細で見える。

2. すでに進めたこと（実装済みの方向）

認証

管理者メールの新規登録（ブートストラップ）・メール正規化・Hosting のキャッシュ対策。

UI

カテゴリ・依頼者→担当者を削除。一覧はシンプルに。議事録→タスクも同じ前提で保存。

体験

タスク名入力のブラウザ履歴抑止など、入力まわりの細かい改善。

3. 次に「やりたい」こと（検討・提案）

シグナルの見直し：納期ベース → コンサル向けラベル

現状の危険・注意・順調は「期限までの日数 × 進捗」からの自動判定。工数・納品型には強いが、コンサルでは期限が緩くても重要・逆に定常などが混ざりやすい。

案：タスクに緊急 / 重要 / 定常（手動またはルール付き）を付け、一覧の色・フィルタの主軸にする。
併用案：納期超過・本日期限は別バッジやアラートとして残し、「優先度」と「日付リスク」の両方を見えるようにする。

決めると実装が速いポイント

  • 優先度は手動選択のみか、デフォルト（例：定常）からの変更か。
  • 緊急と重要の説明をツールチップや1行ヘルプで固定するか（運用ブレ防止）。
  • 既存タスクの移行ルール（一括で「重要」等にするか、未設定を許すか）。
