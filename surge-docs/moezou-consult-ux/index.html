もえぞうタスク｜種別の運用（3番）と UI/UX 案

資料用（Surge 公開）・コンサル向けダッシュボードのイメージ整理

目的。 緊急・重要・定常の運用を助ける機能を入れたうえで、タスクごと・人（担当）ごとに見通しを良くする次の一手を検討するためのメモです。現状のカード型ダッシュボード（集計・フィルタ・担当者バッジ）を前提にしています。

1. 実装した「3番」（種別の運用サポート）

説明

種別の目安。 追加・編集モーダル内で、緊急／重要／定常の使い分けのヒントを折りたたみ（details）で表示。オンボーディングと週次のすり合わせで効く。

レビュー

要レビュー。 「重要」かつ、最終更新（updatedAt、なければ createdAt）から 7日以上動いていない未完了タスクを対象。ツールバーのフィルタとカード上のバッジで一覧できる。コメント投稿時に親タスクの updatedAt を更新し、「触った日」がレビュー判定に反映される。

定常

定常の繰り返し。 定常タスクで「完了時に同内容の次タスクを作る」を有効にし、日数（例: 7）を指定。完了すると期限＋N日で新規タスクが自動作成される。詳細モーダルに一行説明を表示。

2. タスク軸（1タスクを追いやすくする）アイデア

  • 次アクション一行。 タイトル直下に「次にやること」だけを短く表示（メモと分離）。週次で「止まってる理由」が見える。
  • タイムライン（詳細内）。 作成・編集・コメント・進捗変更を時系列で並べ、議事録→タスクの流れとつなげる。
  • ブロック理由タグ。 「クライアント待ち」「社内確認中」などを1タップで付与。要レビューと組み合わせて優先度の再判断がしやすい。
  • サブタスクの「今日やる」。 サブタスクに「今日」マークを付け、ダッシュボード上部にミニリスト（今日の3件）を出す。

3. 人軸（誰のバックログかを見やすくする）アイデア

  • メンバー別スイムレーン。 横スクロールの列＝メンバー、セル＝その人の未完了タスク。ドラッグで担当変更（将来）。現状の「全員」ドロップダウンと両立しやすい。
  • 「自分の今週」ビュー。 フィルタを「自分＋期限7日以内＋未完了」にワンクリック。週次ミーティング用。
  • 負荷インジケータ。 メンバー名の横に、担当件数・緊急件数・要レビュー件数の小さな数字。偏りが一目で分かる。
  • アバターリング。 カードの角に担当者の色付きイニシャル（既存）を複数表示し、共同担当を視覚化。

4. 情報の流れ（概念図）
