# 「正（オリジナル）はどこ？」のルール（SSOT）

## まず一言で

**同じルールを2か所にコピペしておくと、あとで片方だけ直して食い違いが出る。**  
だから「この内容の正本はこのファイル」と決めておく。これを **SSOT（Single Source of Truth）** と呼ぶ。

---

## このSKILLパック内：正本の対応表（必ずここに従う）

| 内容 | 正本（ここだけを編集する） | やってはいけないこと |
|------|---------------------------|----------------------|
| **入口・スコープ・作業の大まかな順番** | `SKILL.md` | 同じ順番説明を `reference-*.md` に長く複製しない |
| **企画〜構成案の対話手順・質問の流れ** | `reference-script-structure-dialogue.md` | 別ファイルに同じフェーズ一覧をコピーしない |
| **PASTOR の略字・意味・表** | `reference-script-structure-dialogue.md` のみ | `SKILL.md` / `reference-pending.md` に PASTOR 表を貼らない |
| **トーン・暫定出力件数・未確定の追記** | `reference-pending.md`（ユーザーが書いた部分） | 確定ルールを口頭だけにしてファイルと食い違わせない |
| **ベルラスHTML資料の長い細則** | 個人SKILL `belle-lus-doc-creator` | YT用フォルダに**全文コピー**しない（参照一文に留める） |

**優先順位**：`reference-pending.md` の**ユーザー確定文** ＞ AIの推測。  
**矛盾**：ファイル同士で食い違ったら **ユーザーに確認**。エージェントが勝手に複数ファイルを別ルールに書き換えて正本を壊さない。

---

## ベルラスと「雰囲気を合わせたい」とき

- **短い方針**だけ、`reference-pending.md` などに書いてよい（例：美容ばかりに寄せない、根拠のない断定をしない）。
- **細かい条文**まで揃えるなら、YT側に長文を貼らず、**「詳細は belle-lus-doc-creator の〇節に合わせる」** と**参照**で済ませる（SSOT 維持）。

---

## AI（エージェント）向けチェックリスト

- PASTOR の **A は Amplify**。`reference-script-structure-dialogue.md` 以外の定義に従わない。
- 手順を変えたいときは **正本ファイル1つ**を直し、他は参照のままにする。
